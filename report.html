Defining the Problem

威利在哪裡？是一本世界知名的繪本，內容為一張張的複雜繪圖中，找到圖中存在的一位人物—威利
由於有些構圖實在是過於複雜，如藏在壅擠的人潮中，或者藏在圖片不起眼的部位，因此本次以機器學習的方式去偵查威利在圖中的位置

本次問題為：在一張已知威利存在的繪本圖當中，找到威利，並用顯眼的方式標記出來

Gathering Data

資料獲取的方式為：在網路上搜尋威利在哪裡，根據google結果，在其中找到幾張解析度較高、較大的圖片下載

其中多數的圖片從這個網站上透過google提供圖片搜尋的功能找到內容相近，且較不模糊的結果

Data Processing

  • 本次的資料與以往不同，是要從一張大圖中，找到符合威利外表的其中一部份，因此換個想法：手動將圖中的威利找出並擷取小圖與其座標，然後在從原本大圖中隨機擷取幾張小圖，為not_wally的部分
  • 觀察資料集中，威利的圖片有特定的規則

    • 無論是側臉或是正臉，威利的臉與路人精細度不一樣、臉形不一樣、髮型不一樣，而且頭上還有一頂幾乎是只有他會戴的帽子，因此在資料的前處理，有些model使用了HOG。

Building Models

  • Selective Search

    用Scan Window會太慢

    • Selective Search是使用segmentation後的結果，套用hierarchical grouping algorithm產生intrest window
    • RGB 每個色彩各取25個bins後標準化
    • 每個區域riri有顏色Ci=c1i...cniCi=ci1...cin

      RGB都使用時n=75n=75

    • 相似度s(ri,rj)=∑min(cki,ckj)s(ri,rj)=∑min(cik,cjk)
    • 簡而言之就是將物體用Selective Search產生初始區域，然後重複計算各區域間相似值，並把相似性高的區塊合併
  • HOG
    • 圖片中的"物件"和周圍會有較大的變化，在離散的資料中，用差分類比微分，即顏色的變化率，分別對圖片兩軸做差分後，平方合開根號即為邊緣強度，然後將兩軸數值取arctan後就可得邊緣方向
    • HOG會把圖片分成數個cell，每個區塊統計某方向的gradient大小方向，分配給不同的角度，然後將不同cell合成一個block並normalize，以減少亮度的影響，再將這些所謂的特徵向量重新拼接成一維向量

      來源
  • KNN (with HOG)
    • 由於我認為所有威利圖片形狀應為一致的，因此不同威利的圖片相似度應比其他還高，這次是將圖片經HOG轉換後，由KNN分別是否為威利
  • SVM (with HOG)
    • 因為網路上有許多HOG搭配SVM使用的範例，且openCV甚至也內建了預先訓練好的SVM(辨識人類等)，因此也作為本次的model之一
  • CNN
    • 對於影像識別，CNN可以說是其中大宗，許多的影像處理都是有CNN的影子在(RCNN、VGG16等)

Conclusion & Application

  • 本次實驗結果可以說是相當失敗？，fail case discuss

    • 出自於威利在原圖中的大小差異過大，在resize同一大小後，差異可以看到不小
